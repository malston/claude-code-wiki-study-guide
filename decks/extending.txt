What are the three extension mechanisms in Claude Code and their primary purposes?	Subagents: task delegation with isolated reasoning<br>Skills: knowledge/workflow injection into main context<br>MCP Servers: external tool/API access (stateless, deterministic)	extending::architecture
What isolation model do subagents use?	Each subagent gets its own context window, separate from the main instance. It starts fresh ("clean slate"), maintains its own conversation transcript, and returns only a summary to the main instance.	extending::subagents
Where do subagent definition files live?	Project: .claude/agents/*.md<br>Personal (cross-project): ~/.claude/agents/*.md	extending::subagents
A task would consume 30+ turns of investigation and you want to protect the main context. Subagent, skill, or MCP server?	Subagent. It gets an isolated context window, so the 30+ turns don't pollute the main conversation. The main instance only sees the returned summary.	extending::subagents extending::scenario
You need Claude to follow your team's REST API naming conventions when designing endpoints. Subagent, skill, or MCP server?	Skill. This is lightweight guidance (knowledge injection) for work happening in the main context. Auto-activates when API work is detected, no isolation needed.	extending::skills extending::scenario
You want Claude to query your Postgres database during conversations. Subagent, skill, or MCP server?	MCP server. Database access is a stateless external tool call -- deterministic, no reasoning needed, no conversation context required.	extending::mcp extending::scenario
In the Lens + Reviewer pattern, which component is the skill and which is the subagent?	Lens = Skill (lightweight, auto-activates during coding, awareness checklist)<br>Reviewer = Subagent (thorough investigation, isolated context, on-demand explicit delegation)	extending::architecture extending::patterns
What are the three YAML frontmatter fields in a SKILL.md file?	name: lowercase identifier (max 64 chars)<br>description: what triggers auto-discovery (max 1024 chars)<br>allowed-tools: (optional) restricts available tools when active	extending::skills
Skills inject two categories of content. What are they?	Knowledge injection: facts, rules, standards (what to know)<br>Workflow injection: processes, sequences, checklists (how to work)	extending::skills
Where can skills be sourced from? List the three sources.	1. Personal: ~/.claude/skills/<br>2. Project: .claude/skills/ (version controlled)<br>3. Plugin: bundled in installed plugins' skills/ directories	extending::skills
Each skill description costs ___ tokens per message in the catalog.	~25-100 tokens per message. Full skill content only loads on invocation.	extending::skills extending::tokens

To run a skill in an isolated subagent context (protecting main conversation from large data reads), set ___ in the SKILL.md frontmatter.	context: fork<br>(Optionally also set agent: to specify which subagent type provides the system prompt)	extending::skills extending::config
What does the !`command` syntax do inside a SKILL.md file?	Dynamic context injection. The shell command executes before the skill content reaches Claude, and its output replaces the !`command` in the skill text. Example: !`gh pr view $ARGUMENTS` injects actual PR data.	extending::skills
What string substitution variables are available in skill content?	$ARGUMENTS: all arguments passed on invocation<br>$ARGUMENTS[N] or $N: specific argument by 0-based index<br>${CLAUDE_SESSION_ID}: current session ID	extending::skills extending::config
In a subagent YAML frontmatter, what does the tools field do vs disallowedTools?	tools: allowlist -- subagent can ONLY use listed tools<br>disallowedTools: denylist -- subagent inherits all tools EXCEPT listed ones<br>Use tools for narrow jobs, disallowedTools when it needs most tools but you want to block specific actions.	extending::subagents extending::config
What are the six subagent permission modes?	default: standard permission checking<br>acceptEdits: auto-accept file edit/write<br>dontAsk: auto-deny all permission prompts<br>delegate: coordination-only (for team leads)<br>bypassPermissions: skip all checks<br>plan: read-only exploration mode	extending::subagents extending::config

What are the three hook types in Claude Code?	Command: runs a shell script (receives JSON on stdin, communicates via exit codes + stdout)<br>Prompt: single-turn LLM evaluation (Haiku by default, returns ok/not ok)<br>Agent: spawns a subagent with multi-turn tool access (up to 50 turns)	extending::hooks

PostToolUse hook with matcher "Edit|Write".<br>Config: {"hooks":{"PostToolUse":[{"matcher":"Edit|Write","hooks":[{"type":"command","command":"jq -r '.tool_input.file_path' | xargs npx prettier --write 2>/dev/null"}]}]}}	extending::hooks extending::scenario
