What is the core problem when using Claude Code on large codebases?	The context window is a fixed token budget. Every file read, conversation turn, and tool result costs tokens. When exhausted, auto-compaction summarizes older content and Claude loses detail -- forgetting instructions, hallucinating paths, re-reading files, and breaking mid-refactor.		guides::large-codebase-strategies

What does the <code>/batch</code> command do and when should you use it?	It decomposes wide, uniform changes into 5-30 independent units and spawns parallel agents -- each with its own context window and git worktree. Use it for dependency migrations, API contract changes across callers, pattern enforcement, or library replacements across many files.	How it works:<br>1. <b>Research and Plan</b> -- orchestrator launches explore agents to find affected files and decompose work<br>2. <b>Spawn Workers</b> -- one background agent per unit, all in parallel with isolated worktrees<br>3. <b>Track Progress</b> -- status table updates as agents complete; each worker implements, tests, and opens a PR<br><br>When <b>not</b> to use it:<br>- Changes with dependencies between units (use Agent Teams)<br>- Surgical changes in one area (single focused session)<br>- Exploratory work where the approach isn't clear<br><br>Key tip: review the plan carefully before approving. If proposed units aren't truly independent, push back -- overlapping units cause merge conflicts.	guides::large-codebase-strategies
